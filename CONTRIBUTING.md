# Contributing

Thank you for your interest in ratingtables!

The project is still in an early stage of development, so APIs and data structures may change.

## Reporting Issues

Please open a GitHub Issue if you find:

- Bugs
- Unexpected behavior
- Documentation errors
- Ideas for new features

When possible, include:

- a reproducible example
- expected behavior
- actual behavior
- sessionInfo()

## Pull Requests

Pull requests are welcome.

Please:

- keep changes focused
- update documentation if needed
- include tests when adding functionality
- avoid introducing unnecessary dependencies

## Design Philosophy

ratingtables aims to be:

- lightweight
- primarily base R
- transparent
- easy to audit
- suitable as a reference implementation for actuarial rating workflows

Design discussions are encouraged before implementing large features.
